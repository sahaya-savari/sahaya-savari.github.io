# Model Context Protocol (MCP) Explained: Architecture, Tools & Python Implementation

As Large Language Models become integral to software development, connecting models to enterprise databases, local codebases, design files, and third-party APIs has historically required custom integration logic for every application.

The **Model Context Protocol (MCP)**, open-sourced by Anthropic, establishes an open, standard client-server protocol enabling LLMs to seamlessly interact with external data sources and tools without rewriting custom middleware wrappers.

---

## 🔌 Architecture Overview

MCP follows a clear Client-Server model over standard I/O (stdio) or Server-Sent Events (SSE HTTP):

```
+------------------+         MCP JSON-RPC Protocol       +--------------------+
|   MCP Client     | <=================================> |   MCP Server       |
| (Claude Desktop/ |   - Prompts / List Tools            | (Database Explorer |
| IDE Extensions)  |   - Call Tool (SQL / File Search)   |  File Manager Server)|
+------------------+                                     +--------------------+
```

### Core Primitives in MCP:
1. **Resources**: URI-addressable data content (e.g. `file:///logs/app.log` or `postgres://table/users`) exposed for model context.
2. **Tools**: Functions callable by the LLM with typed input arguments validated by JSON Schemas.
3. **Prompts**: Reusable pre-configured template workflows exposed by the server.

## 💻 Python Example: Building a FastMCP Server

```python
from pydantic import BaseModel, Field
from mcp.server.fastmcp import FastMCP

# Initialize FastMCP Server
server = FastMCP("System Health MCP Server")

class SystemMetrics(BaseModel):
    cpu_percent: float
    memory_available_gb: float
    status: str

@server.tool()
async def check_system_health() -> str:
    """Returns the current server health and hardware resource utilization."""
    metrics = SystemMetrics(
        cpu_percent=14.2,
        memory_available_gb=12.4,
        status="HEALTHY"
    )
    return metrics.model_dump_json()

@server.tool()
async def query_application_logs(severity: str = Field(description="ERROR, WARN, or INFO")) -> str:
    """Searches local logs filtered by severity level."""
    return f"Found 3 log entries matching severity '{severity.upper()}': All background jobs executed cleanly."

if __name__ == "__main__":
    server.run()
```
